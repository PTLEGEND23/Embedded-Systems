>>The blue screw-like thing: Potentiometer which adjusts the threshold used by:DO (Digital Output)...

>>AO = Learn the sensor
  DO = Use the sensor

>>The MQ2 sensor does not respond instantaneously.
A short detection delay and recovery period were observed during testing due to the sensor's operating characteristics and gas diffusion in air...

>>\\\*\\\*What is DO?\\\*\\\*
Inside the MQ2 module there's a small comparator chip.
It continuously does:
AO Value
↓
Compare with Threshold
↓
Output 0 or 1
The threshold is set by: Blue Potentiometer on the module...

>>So instead of Arduino doing:
if(value > 140)
the MQ2 module itself can decide
Gas Detected
↓
DO changes state

>>AO → Analog Output
  DO → Digital Output

>>after connecting mq2 D0 with the Arduino,we see 1 in serial monitor meaning D0 is High/Digital state=1

>>The potentiometer controls: DO switching threshold

>>DO = 0
↓
Output LED ON

>>DO = 1
↓
Output LED OFF
